# ✨ Email Writer Assistant

A Chrome extension that adds an **AI Reply** button directly inside Gmail. Click it, pick a tone
and length, and get a ready-to-send reply generated by Google's Gemini API — powered by a Spring
Boot backend. A React app is included for testing the backend without opening Gmail.

## How it works

```
Gmail thread
     │  click "AI Reply"
     ▼
Chrome Extension (content script)
     │  reads the email thread from the Gmail DOM
     │  sends { emailContent, tone, length }
     ▼
Spring Boot backend  →  builds a prompt  →  Gemini API (generateContent)
     │
     ▼
Reply text returned to the popup → Copy or Insert into the compose box
```

## Project structure

```
Email Writer Assistant/
├── Email Writer Extention/   Chrome extension (Manifest V3)
│   ├── manifest.json
│   ├── content.js             injects the AI Reply button into Gmail
│   ├── gmail.js                Gmail DOM helpers (compose box, send button, email text)
│   ├── api.js                  talks to the Spring Boot backend
│   ├── popup.js                 popup UI logic (tone/length, generate, copy, insert)
│   └── *.css
├── email-writer-sb/           Spring Boot backend
│   └── src/main/java/com/email/writer/
│       ├── Controller/          REST endpoint
│       ├── Service/               prompt building + Gemini API call
│       └── Model/                  request DTO
└── email-writer-react/        React app for testing the backend directly
```

## Features

- One-click **AI Reply** button injected into Gmail's compose/reply toolbar
- Tone (Professional / Friendly / Casual) and length (Short / Medium / Long) controls
- Copy to clipboard or insert directly into the Gmail compose box, with confirmation feedback
- Regenerate button, live character counter, and a loading state while the reply generates
- Popup closes with `Esc` or by clicking outside it
- Standalone React UI for testing the backend without Gmail — same tone/length controls

## Setup

### 1. Get a Gemini API key

Create a free key at [aistudio.google.com/apikey](https://aistudio.google.com/apikey).

### 2. Run the backend

```bash
cd email-writer-sb
export GEMINI_KEY=your_api_key_here
export GEMINI_URL=https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent
./mvnw spring-boot:run
```

The backend starts on `http://localhost:8080`.
(If you're running this from IntelliJ instead, set `GEMINI_KEY` and `GEMINI_URL` in
**Run → Edit Configurations → Environment variables** rather than exporting them in a terminal.)

### 3. Load the Chrome extension

1. Open `chrome://extensions`
2. Enable **Developer mode**
3. Click **Load unpacked** and select the `Email Writer Extention` folder
4. Open Gmail, open any email, and click the **AI Reply** button that appears next to Send

### 4. (Optional) Run the React testing UI

```bash
cd email-writer-react
npm install
npm run dev
```

Runs on `http://localhost:5173` and calls the backend at `http://localhost:8080`.

## Tech stack

**Backend:** Java 21, Spring Boot 3, Spring WebFlux (WebClient), Jackson, Lombok
**AI:** Google Gemini API (`generateContent`)
**Extension:** Chrome Manifest V3, vanilla JS, `MutationObserver`
**Frontend (testing UI):** React 19, Vite, Axios

## Known limitations / roadmap

- Gmail's DOM structure can change; `gmail.js` isolates all DOM-dependent selectors so only that
  file needs updating if Gmail changes its markup
- No auth on the backend endpoint — fine for local/personal use; would need API-key auth and rate
  limiting before any public deployment
- Currently runs against `localhost:8080`. To make the extension usable by others (e.g. before a
  public Chrome Web Store listing), deploy the backend (Render/Railway free tier work fine), then
  update `API.BASE_URL` in `api.js` and `host_permissions` in `manifest.json` to the deployed URL
